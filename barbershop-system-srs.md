# Statement of Work (SOW)

**โครงการ:** พัฒนาระบบบริหารจัดการร้านตัดผม (Barbershop Management System)

**วันที่เอกสาร:** 17 ธันวาคม 2025

**เวอร์ชัน:** 1.0

**อ้างอิงเอกสาร:** Software Requirement Specification (SRS) v1.0

---

## 1. บทนำ (Introduction)

เอกสารขอบเขตการดำเนินงานฉบับนี้ จัดทำขึ้นเพื่อระบุรายละเอียดงาน สิ่งที่ส่งมอบ และเงื่อนไขข้อตกลงในการพัฒนาระบบบริหารจัดการร้านตัดผม วัตถุประสงค์หลักคือการสร้างแพลตฟอร์มที่เชื่อมโยงการทำงานระหว่างส่วนหน้าบ้าน (ลูกค้า/ช่าง) และส่วนหลังบ้าน (ผู้บริหาร) ให้ครอบคลุมการจองคิวออนไลน์, ระบบสมาชิก (CRM), การชำระเงิน (POS), และการจัดการสต็อกสินค้า เพื่อเพิ่มประสิทธิภาพการดำเนินธุรกิจ

## 2. ขอบเขตของงาน (Scope of Work)

ผู้พัฒนา (Developer/Vendor) จะดำเนินการออกแบบ พัฒนา และติดตั้งระบบ โดยมีรายละเอียดขอบเขตงานดังนี้:

### 2.1 การพัฒนาระบบ (System Development)

อ้างอิงตาม SRS หัวข้อที่ 3 ผู้พัฒนาจะส่งมอบโมดูลหลัก 6 ส่วน ดังนี้:

1. **ระบบสมาชิกและลูกค้าสัมพันธ์ (CRM)**
* พัฒนาระบบจัดเก็บข้อมูลลูกค้าและประวัติการใช้บริการ
* พัฒนาระบบบันทึกรูปภาพ Before/After ทรงผม
* พัฒนาระบบสมาชิก (Tier System) และระบบสะสม/แลกแต้ม


2. **ระบบจองคิวและบริหารจัดการคิว**
* พัฒนาระบบจองผ่าน Web Application และเชื่อมต่อ LINE OA
* พัฒนาระบบจัดการ Walk-in หน้าร้าน และคำนวณเวลารอโดยประมาณ
* พัฒนาระบบแจ้งเตือนผ่าน SMS/LINE และหน้าจอแสดงผลคิว (Queue Display)


3. **ระบบชำระเงินและจุดขาย (POS)**
* พัฒนาระบบคิดเงิน รองรับเงินสด, QR Payment, บัตรเครดิต
* พัฒนาระบบคำนวณส่วนลด ภาษี และตัดสต็อกอัตโนมัติ
* เชื่อมต่อฮาร์ดแวร์: เครื่องพิมพ์ใบเสร็จ, ลิ้นชักเก็บเงิน
* **Feature พิเศษ:** รองรับโหมด Offline กรณีอินเทอร์เน็ตขัดข้อง


4. **ระบบบริหารจัดการช่าง (Staff Management)**
* พัฒนาระบบจัดการตารางงาน (Shift) และสถานะการทำงาน (ว่าง/ไม่ว่าง)
* พัฒนาระบบคำนวณค่าคอมมิชชั่นตามยอดบริการ


5. **ระบบจัดการสต็อกสินค้า (Inventory)**
* พัฒนาระบบรับเข้า/ตัดออก สินค้าและบริการ
* ระบบแจ้งเตือนเมื่อสินค้าใกล้หมด


6. **ระบบรายงานและผู้ดูแลระบบ (Admin & Reports)**
* พัฒนา Dashboard แสดงยอดขายและประสิทธิภาพรายวัน/เดือน
* ระบบจัดการสิทธิ์ผู้ใช้งาน (Admin, Manager, Staff)



### 2.2 การออกแบบและการจัดการ (Design & Management)

* ออกแบบ UI/UX ให้รองรับการใช้งานผ่าน Web Browser, Mobile และ Tablet (Responsive)
* ออกแบบฐานข้อมูล (Database Schema) บน PostgreSQL หรือ MySQL
* จัดเตรียม Server Environment (Linux Base) ให้พร้อมใช้งาน

### 2.3 การเชื่อมต่อระบบภายนอก (External Interfaces)

* เชื่อมต่อ Payment Gateway
* เชื่อมต่อ SMS Gateway และ LINE Messaging API
* เชื่อมต่อ Google Maps API

## 5. ข้อกำหนดทางเทคนิค (Technical Specifications)

ผู้พัฒนาจะจัดทำระบบตามมาตรฐานที่ระบุไว้ใน SRS ดังนี้:

* **Server Side:** Linux OS, Web Server (Nginx/Apache), Database (PostgreSQL/MySQL)
* **Client Side:** Web Technology ที่รองรับ Tablet และ Mobile
* **Performance:** รองรับผู้ใช้พร้อมกัน ≥ 100 users, Response Time ≤ 2-3 วินาที
* **Security:** เข้ารหัสข้อมูลตาม PDPA, มีระบบ Role-based Access

## 6. ความรับผิดชอบของผู้ว่าจ้าง (Client Responsibilities)

เพื่อให้โครงการสำเร็จลุล่วง ผู้ว่าจ้าง (ร้านตัดผม) มีหน้าที่ดังนี้:

1. จัดเตรียม Hardware (Computer, Tablet, Thermal Printer, Scanner, TV) ตามสเปคที่ระบุ
2. ให้ข้อมูลรายละเอียดสินค้า ราคา และโปรโมชั่น เพื่อนำเข้าระบบ
3. จัดเตรียมบัญชี Payment Gateway, SMS Gateway และ LINE Official Account
4. จัดบุคลากรสำหรับทดสอบระบบ (UAT) ตามระยะเวลาที่กำหนด

## 7. เกณฑ์การตรวจรับงาน (Acceptance Criteria)

1. **Functional Test:** ระบบต้องทำงานได้ครบทุกฟังก์ชันตามหัวข้อ 2.1 และ SRS
2. **UAT (User Acceptance Test):** ผู้ใช้งานฝั่งร้านค้าและจำลองลูกค้าต้องสามารถใช้งานระบบได้จริงโดยไม่พบ Critical Bug
3. **Reliability:** ระบบต้องมีความเสถียร Uptime 99.5% ในช่วงทดสอบการใช้งาน
